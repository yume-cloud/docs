Доступ к данным регулируется на стороне сервера по ролям и разрешениям тенанта. Точные эндпоинты управления пользователями смотрите в OpenAPI (CRM, агенты, группы).

<Warning>
  Изменение прав обычно доступно только администраторам. При недостаточных правах API вернёт **`403 Forbidden`**.
</Warning>

## Роли (концептуально)

| Роль | Возможности |
|------|-------------|
| **Администратор** | Полный доступ, настройки, пользователи |
| **Менеджер** | Операции и согласования без администрирования системы |
| **Оператор** | Повседневные операции по клиентам и выдаче |
| **Наблюдатель** | Только чтение |

Фактические имена ролей и матрица могут отличаться — сверяйтесь с продуктом.

## Проверка доступа

При `403` проверьте JWT, роль пользователя и ограничения по объекту (инвентарь, филиал и т.д.).
