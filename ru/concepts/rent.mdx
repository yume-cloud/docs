Аренда в Yume Cloud — это договорённость на период, связывающая клиента, объект (инвентарь / технику) и условия выдачи и возврата. В API детали сделок и инвентаря отражены в ресурсах CRM (`/v1/crm/...`) — используйте OpenAPI для точных путей и полей.

## Жизненный цикл

Типичные состояния проката (концептуально):

```
Создано → Активно → Завершено
                 ↘ Отменено
```

### Переходы статусов

| Из | В | Когда |
|----|---|--------|
| `created` | `active` | Объект передан клиенту |
| `active` | `completed` | Возврат по `end_date` или факту |
| `active` | `cancelled` | Досрочное расторжение |
| `created` | `cancelled` | Отмена до выдачи |

| Статус | Смысл |
|--------|--------|
| `created` | Запись есть, выдачи ещё не было |
| `active` | Объект у клиента в срок аренды |
| `completed` | Период закрыт, возврат оформлен |
| `cancelled` | Аренда отменена |

<Note>
  При переводе в `completed` обычно формируется итоговый документ/счёт. Убедитесь, что реквизиты клиента и тариф актуальны.
</Note>

## Ключевые поля (концептуально)

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | string | Идентификатор записи |
| `status` | string | Текущее состояние |
| `asset_id` | string | Объект аренды |
| `client_id` | string | Клиент |
| `start_date` | string (ISO 8601) | Начало периода |
| `end_date` | string (ISO 8601) | Конец периода |
| `total_amount` | number | Итоговая стоимость |
| `created_at` | string | Создание записи |
| `updated_at` | string | Последнее изменение |

## Связи

### Транзакции

Платежи по аренде отражаются как [транзакции](/ru/concepts/transactions) с привязкой к сделке.

### Документы

Договоры и счета создаются как документы; при закрытии аренды может автоматически формироваться итоговый счёт.

### GPS

Если на объекте включён трекинг, координаты доступны через GPS и интеграции геолокации в спецификации API.
